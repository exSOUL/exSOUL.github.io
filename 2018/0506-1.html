# Mac(High Sierra)でmikutterを動かしてみた

mikutterのTwitterコンシューマキーが凍結されました というブログを見た。
過去にWindowsマシンをデュアルブートにして、Ubuntuをインストールした時に一度だけmikutterを使っていた。
その後はスマホ用クライアントに落ち着き、mikutterは触らなくなっていた。

ただ、OCS NagoyaやOSC Kyotoなどでたまに作者の方のお話を聞いたりしていたので、せっかくなので過去に一度諦めたMacへのmikutterのインストールを再挑戦してみた。

# brew install mikutter がある

どうやら今はbrew install mikutterだけでインストールできるらしい。便利な時代!
ただ、せっかくなのでGithubにあるリポジトリ(のミラー)からcloneして使ってみる。

# やったこと

適当なディレクトリでgit clone https://github.com/mikutter/mikutter.git

必要なgemをインストール

bundle install --path vendor/bundle

ここでエラーが発生。 libidnというライブラリがなかったので、こちらはbrew install libidnでインストール

もう一度bundle installを実施。正常終了し、必要なgemの導入は完了。

いざ起動してみようとすると、mikutter.rb実行時にエラー発生。

`=~': incompatible encoding regexp match (UTF-8 regexp with ASCII-8BIT string) (Encoding::CompatibilityError)

どうやら文字コード関連でおかしくなっている模様。ググる。
mikutterで同様の現象のものを発見。http://matoken.org/blog/2014/09/02/kobo-mikutter/

上記にあったLANG=ja_JP.UTF-8 mikutterで解決した。(以前何かのタイミングでLANG=Cを設定してたのを直した)

これで無事起動するようになった。
一番最初にあげている記事の通り、Twitterにつなごうとするところで落ちる事を確認し、 mikutterでTwitterを使い続けるには を参考にキーを取得し再起動。

今度は確かにTwitterに繋げられるところまで確認できた。

実際にmikutterから投稿したつぶやき
